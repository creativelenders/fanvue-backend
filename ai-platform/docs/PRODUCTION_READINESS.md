# Production readiness checklist

## Required before deployment

- Set `APP_ENV=production`, `API_AUTH_SECRET`, and `FANVUE_WEBHOOK_SECRET` in a secret manager. Startup rejects production configuration without both secrets.
- Apply migrations in order, including `006_webhook_events.sql`.
- Commit the package-manager lockfile generated by your approved Node version, then change CI from `npm install` to `npm ci`.
- Configure `ALLOWED_HOSTS` and explicit `ALLOWED_ORIGINS`; do not use wildcard origins.
- Use a managed PostgreSQL instance, backups, encryption at rest, and a tested restore procedure.
- Terminate TLS before the API and restrict database/Redis access to private networks.

## Operational safeguards

- Keep `AGENT_EXECUTION_ENABLED=false` and `FINANCIAL_ACTIONS_ENABLED=false`. The included code deliberately does not support live command, trading, or cross-chain execution.
- Process approved media jobs through a durable worker queue. Never dispatch media or PPV actions straight from a request handler.
- Monitor webhook rejection rates, duplicate events, queue latency, failed jobs, approval aging, and authentication failures.
- Retain only data required for operations; document deletion/export workflows for subscriber and generated-media records.

## Human review

Media jobs begin in `awaiting_approval`; an owner or admin must approve them before their status becomes `queued`. Sensitive conversations, age/consent concerns, refunds, chargebacks, legal matters, self-harm, or coercion must be routed to trained human operators.
